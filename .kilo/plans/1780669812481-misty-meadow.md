# Plan: Implement T9 Predictive Text System

## Goal
Implement a lightweight, efficient T9 predictive text system for the `TextEntry` component, including a Python-based dictionary generation tool and a high-performance lookup module.

## Implementation Steps

### 1. Dictionary Generation (Python Tooling)
- **File**: `tools/gen_t9_dict.py`
- **Input**: `tools/2000.txt` (frequency-ordered words).
- **Logic**:
    - Map words to 64-bit digit sequences (e.g., "hello" $\rightarrow$ `43556`).
    - Store entries as `(uint64_t digits, uint16_t word_offset)` pairs.
    - Sort entries by `digits` ascending, then by frequency rank (original position) as tiebreaker.
    - The 2000.txt file is sorted by most frequent at top
- **Output**:
    - `src/t9_dict.bin`: 4-byte header (`entry_count`, `string_table_size`), sorted entry array, and a null-terminated string table.
    - `src/t9_dict.h`: `extern const uint8_t t9DictData[]` and header constants.
- **Verification**: Include a self-check pass in the script to validate known word lookups (e.g., `43556` $\rightarrow$ "hello").

### 2. Firmware Lookup Module
- **Files**: `src/Elements/T9Dict.h`, `src/Elements/T9Dict.cpp`
- **Logic**:
    - `init()`: Parse `t9DictData[]` using pointer arithmetic (no heap copies).
    - `bestMatch(uint64_t digits)`: Binary search for the first match.
    - `matchCount(uint64_t digits)`: Return number of contiguous entries with the same key.
    - `matchAt(uint64_t digits, int index)`: Return the $n$-th match in the collision group.
- **Complexity**: $O(\log n + k)$ where $n$ is entry count and $k$ is match count.

### 3. TextEntry Integration
- **Files**: `src/Elements/TextEntry.h`, `src/Elements/TextEntry.cpp`
- **State Management**:
    - Add `bool t9Mode`, `std::string t9Digits`, and `int t9MatchIndex`.
    - Update `capsText` label to show "T9" when in T9 mode.
- **Input Handling**:
    - **`buttonHeld(BTN_R)`**: Toggle `t9Mode`, clear `t9Digits`/`t9MatchIndex`, and update indicator.
    - **`buttonReleased(BTN_R)` (Short Press)**: If `t9Mode`, commit word (add space, reset index/digits) instead of cycling caps.
    - **`keyPress()`**:
        - If `t9Mode`: Map button to digit $\rightarrow$ append to `t9Digits` $\rightarrow$ lookup `bestMatch` $\rightarrow$ update `textarea` with `LV_ANIM_OFF`.
        - Skip existing `currentKey`/`index` logic when in `t9Mode`.
    - **`buttonPressed(BTN_UP/BTN_DOWN)`**:
        - If `t9Mode`: Increment/decrement `t9MatchIndex` (clamped) $\rightarrow$ update `textarea` with `matchAt()`.
    - **`buttonPressed(BTN_L)`**: Pop last char from `t9Digits` and re-run lookup.
- **Auto-Capitalization**: After committing a word, if the previous char was sentence-ending (`.`, `!`, `?`), capitalize the first letter of the new word.
- **Persistence**: Ensure `t9Mode` persists across `stop()`/`start()` calls.
- **Safety**: Guard `cannedMessages` and `setCapsMode()` with `if(!t9Mode)`.

### 4. Cleanup & Testing
- Ensure `loop()` timer returns early if `t9Mode` is active.
- Verify memory usage (Flash-resident lookup).
- Validate against manual word entry in firmware.

## Constraints & Notes
- **No Heap**: Use pointer arithmetic into Flash-resident data for `T9Dict`.
- **Visuals**: Use `LV_ANIM_OFF` for all T9 word swaps to prevent jarring movement.
- **Memory**: Dictionary must stay within a few dozen KB to be suitable for the target hardware.
